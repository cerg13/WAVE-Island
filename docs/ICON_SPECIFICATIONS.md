
**Use:** Google Play Store listing

### App Icons (Generated or Manual)

```
xxxhdpi: 192 x 192 pixels (4.0x)
xxhdpi:  144 x 144 pixels (3.0x)
xhdpi:   96 x 96 pixels   (2.0x)
hdpi:    72 x 72 pixels   (1.5x)
mdpi:    48 x 48 pixels   (1.0x, baseline)
```

**Adaptive Icons (Android 8.0+):**
```
Foreground: 108 x 108 dp canvas
Background: 108 x 108 dp canvas
Safe zone: Center 72 x 72 dp

File structure:
res/mipmap-xxxhdpi/ic_launcher.png
res/mipmap-xxxhdpi/ic_launcher_foreground.png
res/mipmap-xxxhdpi/ic_launcher_background.png
```

### Android Guidelines

âœ… **DO:**
- Use transparency if needed
- Keep safe zone clear (center 66%)
- Design for circular mask
- Test with different shapes
- Export all densities

âŒ **DON'T:**
- Rely on corners (can be cut)
- Use gradients near edges
- Make too complex

## ğŸ“± Platform-Specific Tips

### iOS:

**Background removal:**
iOS adds its own rounded corners, shadow, and gloss. Your icon should:
- Have solid background
- Be full-bleed (use all 1024x1024)
- Look good with 20% corner radius

**Test mask:**
```
Corner radius = 22.37% of icon size
Shadow: Soft, iOS system shadow
```

### Android:

**Adaptive icons:**
Different launchers use different shapes:
- Circle (most common)
- Square
- Rounded square
- Squircle
- Teardrop (Samsung)

**Safe zone:**
Keep important content in center 66% (72x72dp of 108x108dp canvas)

**Test shapes:**
```
res/mipmap-anydpi-v26/ic_launcher.xml:
<adaptive-icon>
    <background android:drawable="@mipmap/ic_launcher_background"/>
    <foreground android:drawable="@mipmap/ic_launcher_foreground"/>
</adaptive-icon>
```
